# Hướng dẫn Xử lý lỗi 404 trong Hệ thống Commics

Dự án Commics sử dụng Astro và Node.js để xây dựng giao diện người dùng. Việc xử lý lỗi 404 (Trang không tìm thấy) được thực hiện một cách nhất quán thông qua các cơ chế sau:

## 1. Trang 404 Trung tâm (`404.astro`)

Tất cả các lỗi không tìm thấy trang đều được điều hướng về file:
`src/ui/src/pages/404.astro`

Đây là trang 404 mặc định của Astro. Khi người dùng truy cập vào một URL không khớp với bất kỳ route nào trong thư mục `src/pages`, Astro sẽ tự động render trang này.

**Đặc điểm:**
- Được tích hợp sẵn vào `Layout.astro` để giữ giao diện đồng nhất.
- Sử dụng mã `robots="noindex, nofollow"` để tránh việc các công cụ tìm kiếm index các trang lỗi.
- Giao diện tối giản, tập trung vào việc hướng dẫn người dùng quay lại trang chủ.

## 2. Điều hướng Cưỡng bức (Explicit Redirects)

Trong các trang dynamic (sử dụng params như `[slug]`), khi dữ liệu fetch từ API trả về rỗng hoặc lỗi, code sẽ thực hiện redirect người dùng về trang `/404`.

**Ví dụ trong `src/ui/src/pages/truyen/[slug].astro`:**
```astro
const { slug } = Astro.params;
let comic;
try {
  comic = await getComic(slug!);
} catch (e) {
  return Astro.redirect('/404');
}

if (!comic) {
  return Astro.redirect('/404');
}
```

Cơ chế này đảm bảo rằng ngay cả khi URL khớp với pattern của một route nhưng dữ liệu thực tế (như ID truyện hoặc tập phim) không tồn tại trong database, người dùng vẫn thấy trang lỗi 404 thay vì một trang trắng hoặc lỗi code.

## 3. Middleware Xử lý URL cũ (`middleware.ts`)

Hệ thống có một lớp Middleware đặt tại `src/ui/src/middleware.ts` để giảm thiểu các lỗi 404 không đáng có.

**Chức năng:**
- Nhận diện các định dạng URL cũ (ví dụ: sử dụng CHAPTER_UUID thay vì slug).
- Tự động gọi API để tìm thông tin chương tương ứng.
- Thực hiện redirect 301 (Permanent Redirect) sang cấu trúc URL mới (`chap-{N}`).
- Nếu không tìm thấy thông tin để redirect, nó sẽ để request tiếp tục và cuối cùng Astro sẽ render trang 404 nếu không có route nào khớp.

## 4. Cấu hình Server (`astro.config.mjs`)

Dự án đang chạy ở chế độ SSR (`output: 'server'`) sử dụng adapter `@astrojs/node`.

Trong chế độ này, việc xử lý 404 diễn ra ở runtime trên server. Mỗi khi nhận được request, server sẽ kiểm tra routing table, nếu không khớp và không có middleware nào handle, nó sẽ trả về response từ `404.astro`.
